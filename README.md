# effectful_mcmc

A bridge from [effectful](https://github.com/BasisResearch/effectful) models to [NumPyro](https://github.com/pyro-ppl/numpyro)'s `NUTS` kernel. Write probabilistic programs as effectful `@defop` calls, then run NUTS over them with one line — and compose mid-program with any effectful handler (intervention, conditioning, custom inference passes) without an MCMC-specific API.

## Install

```bash
pip install -e .
```

Depends on `effectful[numpyro]`, `numpyro >= 0.19`, and `jax >= 0.4`.

## Quickstart

```python
import jax.numpy as jnp
import jax.random as jr

from effectful.handlers.numpyro import Normal, HalfNormal
from effectful_mcmc import sample, MCMC, NUTS

def linear_regression(x, y):
    alpha = sample(Normal(0.0, 10.0))
    beta  = sample(Normal(0.0, 10.0))
    sigma = sample(HalfNormal(2.0))
    sample(Normal(alpha + beta * x, sigma), obs=y)
    return alpha, beta, sigma                      # return per-site handles

mcmc = MCMC(NUTS(linear_regression), num_warmup=500, num_samples=1000)
mcmc.run(jr.PRNGKey(0), x_data, y_data)

# Posteriors are keyed by the Operation behind each sample site;
# `term.op` recovers it. No string names to keep in sync.
alpha, beta, sigma = mcmc.model_return_value
samples = mcmc.get_samples()
print(float(samples[alpha.op].mean()),
      float(samples[beta.op].mean()),
      float(samples[sigma.op].mean()))
```

Prefer string names? Pass `name="alpha"` etc. to each `sample(...)` and use `mcmc.get_samples_by_name()` for an ArviZ-shaped dict. You'll want names anyway for sites you plan to intervene on (`Intervene("sigma", ...)` below).

The full notebook walkthrough — linear regression, hierarchical Bayes (eight schools), and a counterfactual via `Intervene` — is at [`docs/quickstart.ipynb`](docs/quickstart.ipynb).

## Why a bridge

Plain `numpyro.infer.MCMC` works fine on probabilistic programs that look like NumPyro programs. The value of going through effectful primitives is that handler composition then survives MCMC end-to-end:

```python
from effectful.ops.semantics import handler
from effectful_mcmc import Intervene

def counterfactual(x, y):
    with handler(Intervene("sigma", jnp.array(1.0))):
        return linear_regression(x, y)

mcmc_cf = MCMC(NUTS(counterfactual), num_warmup=500, num_samples=1000)
mcmc_cf.run(jr.PRNGKey(0), x_data, y_data)
# 'sigma' is removed from the posterior; alpha/beta posteriors tighten
# because they now condition on the intervention as if observed.
```

`Intervene` is ~10 lines (in `effectful_mcmc/__init__.py`) that intercept the abstract `sample` defop. The same handler works under MCMC, under prior simulation, under SVI, and under any custom inference scheme that routes through `sample`. There's no MCMC-specific intervention API to learn.

Custom handlers follow the same pattern: subclass `ObjectInterpretation`, decorate methods with `@implements(sample)` / `@implements(factor)` / etc., and stack them around the model.

## What's in the package

| Module | Purpose |
|---|---|
| `effectful_mcmc.primitives` | `sample`, `factor`, `param`, `deterministic` defops + `_NumPyroBackend` default handler |
| `effectful_mcmc.compile` | `compile_to_numpyro` — single-trace pass that turns an effectful model into a NumPyro closure |
| `effectful_mcmc.kernels` | `NUTS` facade over `numpyro.infer.NUTS` |
| `effectful_mcmc.mcmc` | `MCMC` driver that wraps `numpyro.infer.MCMC` (lazy compilation, warm-restart support, Operation-handle posterior addressing) |
| `effectful_mcmc.Intervene` | Library-provided handler for the canonical in-trace intervention |

## Distribution coverage

Every distribution registered in `effectful.handlers.numpyro` is exercised through NUTS in `tests/test_distributions.py`. The full compatibility table is regenerated by `scripts/biject_to_spike.py` and lives at [`docs/biject_to_compatibility.md`](docs/biject_to_compatibility.md).

Discrete distributions are intentionally not supported under NUTS (they have no real-line bijection); the test suite pins this as a recorded skip rather than silently passing.

## Kernels

Only `NUTS` is currently exposed. Attempting `from effectful_mcmc import HMC` (or `SA`, `BarkerMH`, ...) raises `ImportError` with a message naming the kernel. Open an issue if you need a kernel and we'll prioritise.
