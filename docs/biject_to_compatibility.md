## Known issues for the bridge to handle

**`Independent`** — `effectful.handlers.numpyro.IndependentTerm`'s 
`support` returns an unevaluated effectful `_CallableTerm` instead of 
the concrete `IndependentConstraint`. The root cause is in 
`effectful/effectful/handlers/numpyro.py` — `_DistributionTerm._is_eager` 
returns `False` whenever any constructor arg is itself a `Term`, even 
when the inner Term is fully concrete (e.g. `Normal(jnp.zeros(2), jnp.ones(2))` 
is a `NormalTerm` with eager arg arrays but still a `Term`).

Bare `numpyro.distributions.Independent(Normal(jnp.zeros(2), jnp.ones(2)), 1).support` 
returns `IndependentConstraint(Real(), 1)` and `biject_to` returns 
`IndependentTransform` correctly. The bridge's compile pass detects 
`_DistributionTerm`s whose `support` is a `_CallableTerm` and recursively 
materializes them via `dist.<cls>(*evaluated_args, **evaluated_kwargs)` 
before computing the constraint. This avoids modifying effectful itself 
for the initial bridge release.

Upstream fix: generalize `_is_eager` to recurse into Term args rather 
than treating any Term as non-eager.
